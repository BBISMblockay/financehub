Summary

Launch Workbench is SILO’s planning surface for product drops and marketing campaigns. One launch container sits on the calendar (by launch date). Everything else — products, channel sends, tasks, assets, links, comments — attaches to that container in a side drawer.

Good for internal pilot: create launches, schedule channel initiatives, track readiness and tasks, filter by designer/status, use calendar + agenda views. Auth and Supabase persistence match the rest of SILO.

Data model

All data lives in Supabase (PostgreSQL). The app reads/writes with the standard anon key + RLS, same as other v2 tools.

ConceptTableWhat it stores
Launch container launch_calendar Title, launch date/time, status, readiness, designer, campaign labels, PO/vendor/SKU refs, hero image URL, notes
Products launch_product_readiness Multiple SKUs per launch: shots, copy, PO, ETA, readiness flags
Channel initiatives launch_channel_items Email, SMS, Meta, site, retail, etc. — each with its own scheduled date/time, owner, status, URL
Tasks launch_tasks Internal checklist items with due date, priority, type, status (open / in progress / done)
Assets launch_assets Named links (Figma, Drive, PDF URLs)
SILO links launch_system_links References to PO, inventory tag, planning scenario, reports, etc.
Comments launch_comments Coordination notes with timestamp
Terminology: In the UI we say Launch container or campaign shell. Initiatives = per-channel scheduled sends (not the same as the free-text “marketing channel” field on the shell, which is a summary label).

Workflow

Recommended flow for marketing and creative. Numbers match the diagram below.

flowchart LR
  A[1. New Launch] --> B[2. Save shell\nlaunch date on calendar]
  B --> C[3. Open drawer]
  C --> D[4. Products\nreadiness]
  C --> E[5. Initiatives\nemail SMS social]
  C --> F[6. Tasks]
  C --> G[7. Assets and links]
  D --> H[8. Run calendar / agenda\nfilters and KPIs]
  E --> H
  F --> H
  1. Create launch — Click + NEW LAUNCH. Optionally expand Optional prefill and search an open PO or existing Silo product to fill title, PO, vendor, units.
  2. Fill launch shell — Title, launch date (this is what appears on the month calendar), time, status, readiness, designer, campaign type. Add hero image (upload or URL).
  3. Save — Launch chip appears on the calendar on the launch date.
  4. Open drawer — Click the chip or an agenda row.
  5. Products tab — Add one row per SKU/line: product shots, copy, PO, ETA, readiness (use for multi-style drops).
  6. Initiatives tab — Add each channel touch (e.g. teaser email Tuesday 8am, main email Thursday 9am, SMS). Each initiative has its own date and time.
  7. Tasks tab — Internal to-dos; change status inline (open → in progress → done).
  8. Assets / Links / Comments — Paste creative URLs, link to SILO PO or planning tools, leave coordination notes.
  9. Operate — Use Calendar for the month view, Agenda for the next ~2 weeks, filters for designer/status/readiness, KPI band for at-a-glance counts.
Calendar nuance (WIP): The month grid only plots launch dates from launch_calendar. Channel initiative dates are stored and visible in the Initiatives tab and KPIs, but do not yet appear as separate chips on the calendar. See Roadmap.

Roadmap

Prioritized from marketing feedback and internal pilot needs. Not calendar-dated — ordered by impact.

Now (pilot blockers / high value)

  • Show channel initiatives on the calendar (secondary chips or dots on initiative scheduled_date)
  • Edit/delete for initiatives, products, and assets
  • Dropdown vocab for launch type and campaign type (reduce free-text drift)

Next

  • Duplicate launch (copy shell + optional children)
  • Agenda filter: “initiatives this week”
  • Export agenda range to CSV
  • File upload on Assets tab (same pattern as hero image)

Later

  • Marketing-scoped RLS policies
  • Approval workflow (draft → approved → live)
  • Task due-date reminders (email/Slack)
  • Launch templates (auto-spawn standard task checklist)
  • Optional iCal feed for launch + initiative dates

Marketing pilot checklist

  • Confirm Supabase tables exist and RLS allows marketing users to read/write launch tables
  • Confirm launch-images bucket for hero uploads
  • Confirm v_launch_po_product_lookup view if using PO prefill
  • Pilot users: marketing lead + creative + ecom (3–5 people)
  • Rule: every drop gets a launch container; every email/SMS gets an Initiative row with date and owner
  • Support: one contact + screenshot + page URL for issues
