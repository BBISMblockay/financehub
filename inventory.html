PO Strategy
Replenishment keeps you in-stock. Growth adds intentional uplift for launches/marketing/wholesale pushes.
How it works: Suggested PO = (TargetCover + SafetyDays) × Avg/Day × GrowthFactor minus On Hand. Hybrid applies GrowthFactor only to rows you tag.
Tags & Drivers
These are lightweight “intent” tags stored in localStorage. Later we can push to Supabase notes/actions.
Company aggregation key (for blanks)
Company View groups by SKU when present. If SKU is blank it falls back to Barcode, then Product+Variant.
Suggestion behavior
After you run Suggest PO, suggestions stay “live” as you change filters/sort/mode.
Rollout tip: Start with Replenish-only for trust. Then add tags for just a handful of “growth bets” each week.
Sales merge
Each location can define a sales_summary CSV to override velocity and add 7/30/90/120 + last sold.
Growth vs replenish
“Replenish” is standard practice. “Growth” is how you drive upside instead of just replacing units sold.
